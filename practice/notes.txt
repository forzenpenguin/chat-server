## Postfix & Prefix
    1. Postfix will increment or decrement the value but the new value will be visible in the next time we use the value not the same time you're incrementing, e.g "cout << num1++ << endl;".
    2. Prefix will increment or decrement the value right away and you can use it immediately, for example "cout << ++num1 <<endl;".

## Polymorphism

        class shape {
        protected: 
            string desc;
        public:
            shape() = default;
            shape(string desc_param)
                desc = desc_param;
            virtual ~shape() = default; // that is a virtual destructor, it is used to ensure that the destructor of the derived class is called when you delete an object of the derived class using a pointer to the base class.
            virtual void show() override {
                cout << "shape: " << desc << endl;
            }
        };

        class oval : public {
        protected:
            double x_radius;
            double y_radius;
        public:
            oval() = default;
            oval(double x_param, double y_param, string desc_param) : shape(desc_param), x_radius(x_param), y_radius(y_param) {};
            virtual ~oval() = default; // that is a virtual destructor, it is used to ensure that the destructor of the derived class is called when you delete an object of the derived class using a pointer to the base class.
            virtual void show() override { ### -> "VIRTUAL" keyword is used to tell the compiler that this function can be overridden in the derived class, and if you call this function using a pointer or reference to the base class it will call the derived class version of the function if it exists.
                cout << "shape: " << desc << endl;
            }
            void do_non_virtual_object() {
                cout << "this is a non virtual function in the oval class" << endl;
        };

        class circle : public oval {
        protected:
            double radius;
        public:
            circle() = default;
            circle(double x_param, double y_param, string desc_param) oval(x_param, y_param, desc_param) {};
            virtual ~circle() = default; // that is a virtual destructor, it is used to ensure that the destructor of the derived class is called when you delete an object of the derived class using a pointer to the base class.
            virtual void show() override { ### -> "OVERRIDE" keyword is used to tell the compiler, catch any error if you misspelled the function name or you didn't use the same parameters as the base class function.
                cout << "shape: " << desc << endl;
            }
        };
        void show_shape(shape* shape) {
            shape->show(); // it will call the show function in the derived class if it exists, otherwise it will call the show function in the base class.
        }
        int main () {
            shape* shape1 = new shape("shape 1");
            shape* oval1 = new oval(4.5, 3.2, "oval 1");
            shape* circle1 = new circle(4.5, 4.5, "circle 1");
            shape1->show(); // it will call the show function in the shape class.
            oval1->show(); // it will call the show function in the oval class.
            circle1->show(); // it will call the show function in the circle class.
            delete shape1;
            delete oval1;
            delete circle1;
            shape* collection[3] {shape1, oval1, circle1}; // that is how to create an array of pointers to the base class and store the derived class objects in it.
            for(shape* shape : collection) {
                shape->show(); // it will call the show function in the derived class if it exists, otherwise it will call the show function in the base class.
            }
            show_shape(shape1); // it will call the show function in the shape class.
        }


## Dynamic_cast<> ();
    1. It is used to safely downcast a pointer or reference to a base class to a pointer or reference to a derived class, it returns a null pointer if the cast is not possible. It's used to print non virtual functions in the derived class when you have a pointer to the base class, it is used to check if the object is of a specific type before calling a function that is specific to that type.
    2. Syntax:
        shape* shape1 = new shape("shape 1");
        oval* oval1 = dynamic_cast<oval*>(shape1); // that is how to use dynamic_cast to downcast a pointer to the base class to a pointer to the derived class, it will return a null pointer since shape1 is not an oval.
        if(oval1) {
            oval1->do_non_virtual_object(); // it will call the non-virtual function in the oval class if the cast is successful.
        } else {
            cout << "shape1 is not an oval" << endl; // it will print this message since the cast is not successful.
        }

## Smart Pointers:
    1. A smart pointer is a class template that provides automatic memory management for dynamically allocated objects, it automatically deallocates the memory when it is no longer needed, it is part of the C++ Standard Library and provides a convenient way to manage dynamic memory without having to manually delete it.
    2. Syntax:
        #include <memory>
        class entity {
            public:
                string name;
                int ID;
        };
        unique_ptr<entity> entity_ptr = make_unique<entity>(); // that is how to declare a unique pointer to an entity, it takes ownership of the dynamically allocated entity and will automatically delete it when it goes out of scope.
        shared_ptr<entity> ptr2 = make_shared<entity>(); // that is how to declare a shared pointer to an entity, it allows multiple pointers to share ownership of the same dynamically allocated entity and will automatically delete it when the last pointer goes out of scope.
        weak_ptr<entity> ptr3(ptr2); // that is how to declare a weak pointer to an entity, it does not take ownership of the dynamically allocated entity and can be used to check if the object still exists before accessing it.
    3. Explanation:
        - unique_ptr is used when you want to have a single owner of the dynamically allocated object, it cannot be copied or shared with other pointers, it is useful for managing resources that should not be shared, such as file handles or network connections.
        - shared_ptr is used when you want to have multiple owners of the dynamically allocated object, it keeps track of the number of pointers that share ownership and will automatically delete the object when the last pointer goes out of scope, it is useful for managing resources that can be shared, such as objects in a game or nodes in a graph.
        - weak_ptr is used to break circular references between shared_ptrs, it allows you to check if the object still exists before accessing it, it is useful for managing resources that can be shared but may have a limited lifetime, such as cache entries or observers in an observer pattern.

## DAYTIME CLIENT:
using boost::asio::ip::tcp;
int main(int argc, char* argv[])
{
	try
	{
		if (argc != 2)
		{
			cerr << "Usage: client <host>" << endl;
			return 1;
		}
		boost::asio::io_context io;
		tcp::resolver resolver(io);
		tcp::resolver::results_type endpoints = resolver.resolve(argv[1], "daytime");
		tcp::socket socket(io);
		boost::asio::connect(socket, endpoints);
		for (;;) {
			vector<char> buf(128);
			boost::system::error_code error;
			size_t len = socket.read_some(boost::asio::buffer(buf), error);
			if(error == boost::asio::error::eof)
				break;
			else if(error)
				throw boost::system::system_error(error);
			cout.write(buf.data(), len);
		}
	}
	catch (std::exception& e)
	{
		std::cerr << "Exception: " << e.what() << "\n";
	}
}
